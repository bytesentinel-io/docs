## Easy installation

`dynami` is available as a Python package on PyPI.org and can be installed with the following command:

To download and use `dynami` via `pip`, python must be installed first:

```shell copy
# Linux
sudo apt-get install python3 python3-pip python3-venv python3-dev -y

# macOS
brew install python@3.11
```

The installation is done via the Python integrated tool `pip`:

```shell copy
python3 -m pip install dynami
```

For `dynami` to be called directly as such, the Python binary must be added as PATH:

```bash filename=".zshrc" copy showLineNumbers
# Linux
export PATH=”$PATH:/usr/local/bin/python”
# macOS
export PATH="$HOME/Library/Python/3.11/bin:$PATH"
```

*Depending on the OS, the installation may require additional parameters.*

## Manual installation

For system administrators and the use in companies we recommend the manual installation way.

<Steps>

### Installation

Python must be installed for the installation. `dynami` does not require a Python version, but we always recommend to install the latest version.

<Tabs items={["Windows", "Linux", "macOS"]}>
    <Tab>
        ```shell copy
        pip install dynami
        ```
    </Tab>
    <Tab>
        ```shell copy
        pip install dynami
        ```
    </Tab>
    <Tab>
        ```shell copy
        pip install --user dynami
        ```
    </Tab>
</Tabs>

### Configuration

To configure `dynami` we create a new python file (e.g. `updater.py`) and write there e.g. the following code snippet:

```python filename="updater.py" copy showLineNumbers
from dynami.ext import Client
# e.g. Hetzner
from dynami.provider import Hetzner
```

### Cronjob

With a cronjob, the created python script can be executed automatically at a specified time. On a Linux system, this can be created as follows:

**Start cronjob processing with the following command:**

```shell copy
crontab -e
```

**Add the following line to the open document:**

```shell copy
*/2 * * * * python3 updater.py >/dev/null 2>&1
```

**If the document was opened with** `nano`**, then simply with** `ctrl` **+** `x` **crontab can be saved.**

### Test

To ensure that the `updater.py` works completely, the script must be executed once manually.

```shell copy
python3 updater.py
```

</Steps>
